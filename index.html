The Solution

A CMDB built for OT/ICS that runs on Google Workspace.

12,000+ assets. One pane of glass. Zero infrastructure.

Asset Inventory

Every device in one view—normalized, classified, de-duplicated. Filter by site, zone, Purdue level, risk band, vendor, protocol. Search by IP, hostname, CLLI, MAC, serial number. 130+ fields per asset including firmware, protocol, ports, credentials, IEC posture, risk drivers, incident history.

Network Config Import

Paste a Cisco IOS, Palo Alto PAN-OS, or Cisco ASA configuration. The CMDB extracts interfaces, subnets, VLANs, ACLs, firewall rules, and credentials—then enriches matched assets automatically. Raw configs never leave the browser. Plaintext secrets are hashed client-side with SHA-256 before they touch the server.

IEC 62443 Posture

Security Level Target vs Achieved per zone. Authentication mechanism coverage. Encryption coverage. Default credential status. Gap KPIs by zone, Purdue level, and business service—computed from your real data, not a template.

Risk Scoring

0–100 risk score decomposed into business priority, exposure, weakness, threat, and controls. Every score is explainable—you can see exactly why an asset is rated the way it is. Telecom-aware weighting (NEC/SAP/TAP site multipliers, OT protocol risk factors).

Data Quality + Incident Tracking

Six automated metrics: duplicate rate, orphaned relationships, verification coverage, stale records, schema drift, sync issues. Run before or after any change. Export for auditors. Log incidents and changes against any asset—per-asset rollups show open incident count, last incident date, last change ID. Linked by UID. Separate sheets—no grid bloat.

FAQ

Frequently asked questions.

Technical

Google Apps Script is Google's serverless JavaScript platform. It's been in production since 2009, powers Gmail Add-ons, Google Workspace automations, and is used by millions of businesses. It has SLA-backed uptime, runs in Google's data centers, and inherits Google's SOC2/ISO27001/FedRAMP certifications. Your script executes on Google's servers—no infrastructure for you to manage.
6-minute execution cap per function call (we use chunked processing for large operations), 10 million cells per spreadsheet (we use external spreadsheets for the master view), and 20 deployment versions (we prune old versions). These are real constraints but well within the range of 5,000–15,000 asset environments.
Yes. The production deployment manages 11,487 real assets. The chunked rebuild system and cold-start guard handle large datasets within the 6-minute cap.
All code is in standard JavaScript (V8 engine). If Google deprecates something, the code can be ported to any JavaScript runtime. The data is in Google Sheets (exportable to CSV at any time). There's no lock-in beyond the effort of re-platforming—which is true of any SaaS.
Plaintext secrets never leave the browser. SNMP communities, BGP keys, IPsec PSKs, and Type-7 passwords are hashed client-side with SHA-256 before transmission. The server stores only one-way fingerprints and boolean flags. Sensitive fields are admin-gated and redacted in the audit log.
The CMDB has a REST API (token-authenticated) for reading inventory data and triggering writes. External systems can pull asset data, push updates, and trigger admin operations. We configure integrations during deployment (Pro/Enterprise tiers).
